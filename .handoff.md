## 4. FAILED PATHS / GOTCHAS (save the next agent time)

- **The goods are an ABSTRACT layer.** `intermediateGoodStocks` feeds *only*
  `supplyChainHealth` (Ôćĺ the drag) and the two secondary effects (pharmaÔćĺplague,
  electronicsÔćĺresearch). Nothing reads goods/stocks into GDP, prices, or trade.
  So expanding the *count* deepens the cascade graph but doesn't yet make goods
  "do more" economically ÔÇö that's deliberate groundwork, not the payoff.
- **The drag is binary / catastrophe-only.** A raw flows iff its extracting
  sector's total output > 0, so the cascade + drag only fire on a *total sector
  shutdown* (deep depression / war) **or an explicit embargo** (now the oil
  shock). The *untouched* raws are still binary; graded availability (┬ž5) is the
  remaining move to make ordinary shortages ÔÇö not just embargoes ÔÇö bite.
- **Verify economy changes with a GOLDEN snapshot, not just the determinism
  test.** `region-longrun`'s determinism test is same-seed self-consistency ÔÇö it
  won't catch a real behaviour shift. A 5-seed ├Ś N-epoch before/after snapshot of
  GDP/treasury/pop/inflation is the tool (it caught a phantom year-boundary shock
  in #277). For the goods expansion, snapshot the **economy only** (omit health/
  stocks ÔÇö those legitimately change with the richer graph).
- **Adding a good that needs an unproxied raw breaks byte-identity.** Any new
  good's inputs must ground out in raws that `rawAvailable` returns true for in
  healthy play (i.e. in `AGRICULTURAL_RAWS`/`EXTRACTIVE_RAWS`), or it's perpetually
  disrupted Ôćĺ health < baseline Ôćĺ the drag fires in healthy play Ôćĺ economy shifts.
- **Keep the chain switching on at 1920.** `START_YEAR` is 1919; goods unlocking
  at 1919 would be active at founding (where a fresh, un-ticked sim has 0 sector
  output Ôćĺ raws read unavailable Ôćĺ fresh-sim unit tests + founding state shift).
- **`npm install`:** `ELECTRON_SKIP_BINARY_DOWNLOAD=1 PLAYWRIGHT_SKIP_BROWSER_DOWNLOAD=1
  npm install --ignore-scripts --no-audit --no-fund` (electron's postinstall 403s
  in this web env and rolls back the whole install).
- **Asset/audio generation** still network-blocked (`huggingface.co` 403, no
  sharp/ffmpeg). Pipeline plumbing is done + dry-run-verified; generate locally.
