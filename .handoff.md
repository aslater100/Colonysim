## 2. COMPLETED (this session — PR-3 slice 1)

User said "continue" → picked up the explicitly-sequenced next step (PR-3) and shipped its
**first, macro-neutral slice** — the #288 follow-on the prior baton named as PR-3's entry:

- **"Goods ride the rails" — trade-route shipments now carry REAL physical cargo.**
  Until now a `tradeFlows` entry recorded a `goodId`+`volume` that were **decorative** —
  only the arbitrage *profit* (`pendingIncome`) ever moved; the cargo itself never left a
  warehouse. Now each flow carries **`cargo`** (a new serialized field): the real units of
  `goodId` **debited from the source town's `goodStocks` on dispatch** (`shipGoodFrom` —
  ships `min(volume, what the town actually holds)`, never negative) and **credited to the
  destination town's `goodStocks` on arrival** (after `transitDays`). **A severed route now
  strands REAL cargo** — the units were debited on dispatch and are simply never credited,
  so they're destroyed (matches the #288 narrative literally, not just "abstract profit
  lost").
- **Macro-neutral — PROVEN by direct diff.** The dispatch decision, transit time and
  `pendingIncome` are **untouched** — the cargo is purely additive per-town bookkeeping.
  Nothing reads intermediate-good *stock magnitudes* into the macro economy (the solver's
  `rawSupplyLevel` proxies **true raws** off sector output and resolves intermediates
  through the **graph**, never their stock size — the same "ledger is gameplay-inert in
  healthy play" insight that de-risked #292/#294). Confirmed empirically: stashed the
  change and re-ran the **8-seed × 181y headless sweep** — the output (GDP/treasury/
  inflation/pop/sat/outcome for all 8 seeds) is **byte-for-byte identical**.
- **Serialization.** `cargo` rides the existing `tradeFlows` serialize; deserialize
  backfills `cargo: f.cargo ?? 0` (old/pre-cargo flows — their goodId/volume were
  decorative — start at 0), mirroring the `pendingIncome` backfill.
- **Why ship this slice first (not full PR-3).** It's the substrate the *real* balance
  change (per-town supply solving) consumes, but it's independently coherent, fully
  verifiable without a human, and macro-neutral — so it lands de-risked, the same
  decompose-and-verify discipline as the #292 seam → #294 storage swap.

## 4. FAILED PATHS / GOTCHAS (save the next agent time)

- **`tradeFlows[].cargo` is the physical-units field** (units of `goodId` in transit);
  `pendingIncome` is the separate arbitrage **profit**. Dispatch debits the **source**
  town (`buySide`, the lower-wage end), arrival credits the **destination** (`sellSide`).
  A hand-built test flow needs a `cargo` (use `0` if you only care about the profit path).
- **Moving intermediate stock between towns is gameplay-inert TODAY** — the macro headless
  is byte-identical with/without it (verified). It only starts to *matter* once the supply
  solve goes per-town (PR-3 slice 2). Don't expect a balance shift from this slice alone.
- **`Settlement.goodStocks` is THE store** (the nation-wide `intermediateGoodStocks` field
  is **deleted**). Go through the accessors (`goodStock`/`produceGood`/`drawGood`/
  `shipGoodFrom`/…); to seed a town directly in a test, poke `(s.goodStocks ??= {})['id']`.
- **`goodStocksSnapshot()` is a DERIVED aggregate** (sum across towns), not the serialized
  form — each settlement serializes its own `goodStocks` via the `...s` spread.
  `restoreGoodStocks` is **legacy-migration only** (old top-level pool → capital, guarded
  so a populated sim isn't clobbered).
- **The supply solver only ever queries `rawLevel` for TRUE raws** (coal/iron/…), which are
  never in the ledger → it reads the **sector proxy**, never a tracked good's magnitude.
  This is *why* moving/destroying intermediate units can't perturb the economy. Preserve
  it: don't make the solve read intermediate stock sizes without re-baselining.
- **`auxRng` is a THIRD serialized RNG** (notable health, loan ids) — separate from
  `rng`/`aiRng`. New incidental stochastic detail belongs on `auxRng`. (`advisorForecast`
  keeps `Math.random` on purpose — UI-only.)
- **The two gates for any `region.ts` economy change:** the **determinism harness**
  (`tests/serialize-determinism.test.ts`, full `serialize()` across same-seed runs) and the
  **save-size guard** (`tests/save-size.test.ts`, century ceiling 192 KiB, ~82 today). For a
  "macro-neutral" claim, ALSO stash-and-diff the headless sweep — that's what proved this
  slice neutral. The future 44-good catalog × N towns is exactly what the save-size guard
  watches — re-check it then.
- **C1 extraction recipe (proven 3×):** move the body verbatim (`this.`→`r.`) into
  `systems/<name>.ts` as `export function tick<Name>(r: RegionSim)`, `import type
  { RegionSim }`, `tick()` calls it. Needs every touched `r.x` PUBLIC. Verify with the harness.
- **`DAYS_PER_MONTH = 5`** (≈60-day year ÷ 12). A `tick()` is sub-day; ~1952 ticks/game-year,
  so loop `while (r.year < target) r.tick()` rather than counting ticks.
- **`npm install`:** `ELECTRON_SKIP_BINARY_DOWNLOAD=1 PLAYWRIGHT_SKIP_BROWSER_DOWNLOAD=1
  npm install --ignore-scripts --no-audit --no-fund`. **Headless sweep:** `npm run sim --
  181 8`. **Bench:** `npx tsx scripts/bench-region.ts`. **tests/** are NOT type-checked by
  the build (`tsconfig` `include: ["src"]`; vitest runs them via esbuild) — `tsc --noEmit`
  only guards `src/`.
