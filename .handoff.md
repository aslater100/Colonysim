## 4. FAILED PATHS / GOTCHAS (save the next agent time)

- **C1 extraction recipe (now proven 4×):** move the body verbatim (`this.`→`r.`) into
  `systems/<name>.ts` as `export function tick<Name>(r: RegionSim)`, `import type
  { RegionSim }`, `tick()` calls it; make every touched `r.x` PUBLIC; update the tests that
  called `r.fn(…)` to import & call `fn(r, …)` (the method is *removed*, not wrapped — a
  wrapper would name-clash the import). Verify with the determinism harness + a headless byte-diff.
- **`systems/arbitrage.ts` imports `INTERMEDIATE_GOODS` (a *value*) from `region.ts`** while
  `region.ts` imports `tickPriceArbitrage` back — a runtime import cycle. It's **safe**
  because the value is only read *inside the function body* (call-time), never at module top
  level, so ESM live-bindings have it initialized by the time a tick runs (confirmed: harness
  + headless + full suite all green). Keep new `systems/` value-imports call-time-only, or
  move the catalog to its own module if a top-level use ever appears.
- **`tradeFlows[].cargo` is the physical-units field** (units of `goodId` in transit);
  `pendingIncome` is the separate arbitrage **profit**. Dispatch debits the **source** town
  (`buySide`, lower-wage end) via `shipGoodFrom`; arrival credits the **destination**
  (`sellSide`) via `addGoodStock`. A hand-built test flow needs a `cargo` (use `0` for the
  profit-only path).
- **Moving intermediate stock between towns is gameplay-inert TODAY** — the macro headless is
  byte-identical with/without it (verified twice now). It only starts to *matter* once the
  supply solve goes per-town (PR-3 slice 2). Don't expect a balance shift before then.
- **`Settlement.goodStocks` is THE store** (nation-wide `intermediateGoodStocks` is deleted).
  Go through the accessors (`goodStock`/`produceGood`/`drawGood`/`shipGoodFrom`/`addGoodStock`);
  to seed a town directly in a test, poke `(s.goodStocks ??= {})['id']`.
- **The supply solver only ever queries `rawLevel` for TRUE raws** (coal/iron/…), never in
  the ledger → it reads the **sector proxy**, never a tracked good's magnitude. This is *why*
  moving/destroying intermediate units can't perturb the economy. PR-3 slice 2 is exactly the
  step that breaks this on purpose (per-town intermediate stock starts gating production) — an
  **intentional** divergence; re-baseline then.
- **`auxRng` is a THIRD serialized RNG** (notable health, loan ids) — separate from
  `rng`/`aiRng`. New incidental stochastic detail belongs on `auxRng`. (`advisorForecast`
  keeps `Math.random` on purpose — UI-only.)
- **The two gates for any `region.ts` economy change:** the **determinism harness**
  (`tests/serialize-determinism.test.ts`) and the **save-size guard** (`tests/save-size.test.ts`,
  century ceiling 192 KiB, ~82 today). For a "macro-neutral" claim, ALSO stash-and-diff the
  headless sweep. The future 44-good catalog × N towns is what the save-size guard watches —
  re-check it then.
- **`DAYS_PER_MONTH = 5`** (≈60-day year ÷ 12). A `tick()` is sub-day; ~1952 ticks/game-year,
  so loop `while (r.year < target) r.tick()` rather than counting ticks.
