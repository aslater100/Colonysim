## 2. COMPLETED (this session — Spatial Phase C: per-hex terrain yields + building adjacency)

User said "Continue" → the spatial track's headline next step was **Phase C** (terrain
yields feed the economy, the natural activator of PR-3 slice 3's dormant coupling).

- **`tileYieldFor(t: Settlement)`** (private, in `region.ts`). Iterates all ring cells
  (d=1..CITY_WORK_RADIUS, skipping water / center) and computes:
  - `agriculture`: `(meanFertility − 1.0) × 0.20 + (riverFrac) × 0.06`, clamped [-0.15, 0.25]
  - `industry`: `(roughFrac × 0.10) + (oreFrac × 0.15) + (forestFrac × 0.06)`, clamped [-0.05, 0.25]
  - `services`: `(riverFrac × 0.10) + (coastal ? 0.08 : 0)`, clamped [0, 0.20]
  - `information`: 0 (reserved for later eras)
  Result is cached per settlement-id on `_tileYieldCache: Map<number, ...>` — transient
  private field, NOT serialized (cache is rebuilt lazily on first call after load).
- **`placedBuildingTerrainBonus(t, sector)`** (private). For each placed building whose
  `def.sector === sector`, checks cell terrain: agri building on fertile/river cell → +5%;
  industry building on ore/rough cell → +5%; services building on river/coastal → +5%.
- **`buildingBonus(t, sector)`** extended to add both — `yields[sector]` + adjacency bonus.
  `updateSectors` unchanged (already consumes `buildingBonus`).
- **17 new tests** in `tests/terrain-yields.test.ts`: bounds/finiteness (5 seeds × 4 sectors),
  coastal bonus (seed 2 + `pref: 'coastal'`), cache stability (reference equality + determinism),
  adjacency fires/doesn't-fire rules, integration (output > 0 after 1 month, distinct seeds
  differ, fertile > neutral placement), serialize round-trip, serialization discipline
  (`_tileYieldCache` NOT in JSON).
- **Gates:** tsc clean (only pre-existing node-types error); **1020 tests** (1003 + 17);
  determinism harness ✅; bench-region PASS (worst tick 11.9ms < 16.7ms); vite build green;
  headless 8-seed × 181y stable (all values finite, inflation 2.0%).
- **Key gotcha in tests:** `advanceMonth` must be 1440 ticks (30 days × 48 ticks/day),
  NOT 30 — `monthlyUpdate()` fires on day%30===0, not month. Coastal start: seed 2 +
  `{ pref: 'coastal' }` (foundColony option key is `pref`, not `terrain`).

## 4. FAILED PATHS / GOTCHAS (save the next agent time)

- **`advanceMonth` = 1440 ticks, not 30.** `monthlyUpdate()` fires every 30 days; each day
  is 1440÷30 = 48 ticks (REGION_MINUTES_PER_TICK=30, MINUTES_PER_DAY=1440). A 30-tick run
  is less than 1 day — `updateSectors` never fires → sector outputs are 0.
- **`foundColony` option key is `pref`, NOT `terrain`.** The opts signature:
  `{ pref?: 'river-valley' | 'coastal' | 'highlands' | 'surprise' }`.
  Seeds 3..42 don't produce coastal starts without the pref; seed 2 + `pref: 'coastal'` does.
- **`_tileYieldCache` is transient.** Do NOT add it to serialize/deserialize. Cache is rebuilt
  on first `tileYieldFor` call — determinism is preserved because terrain is static (seed-deterministic).
- **Phase C is an intentional re-baseline.** Headless markers WILL differ from the slice-3 base
  (the prior "byte-identical" baseline). Don't "fix" the deviation — it's the terrain shaping
  economies. The headless suite still passes (stable + finite + bounded), just at shifted values.
- **Phase C activates slice 3's dormant coupling.** When towns specialise (agri town produces no
  industry goods and vice-versa), `localGoodsScarcity` > 0, and cost-push + output drag fire.
  After Phase C lands, re-run the economy-balance tests with a playtest (depression ~1929, war).
  The gate-driven (not stock-magnitude) design stays correct — don't switch it.
- **Two price notions, deliberately distinct.** ARBITRAGE price (`localGoodPrice`) = stock/demand
  based; MACRO index (`localGoodsScarcity`) = gate based. They agree under specialisation, differ
  under a raw shock (gate stays 0 for self-sufficient towns, stock would not).
- **C1 extraction recipe (proven 5×) + the systems/region.ts import cycle** are unchanged.
  Keep new `systems/` value-imports call-time-only. `tradeFlows[].cargo` is the physical-units
  field; dispatch debits the SOURCE, arrival credits the destination.
- **`Settlement.goodStocks` is THE store**; go through the accessors. To seed a town directly in a
  test, poke `(s.goodStocks ??= {})['id']`.
- **`auxRng` is a THIRD serialized RNG** (notable health, loan ids). New incidental stochastic
  detail belongs on `auxRng`.
- **The two gates for any `region.ts` economy change:** the determinism harness
  (`tests/serialize-determinism.test.ts`) and the save-size guard (`tests/save-size.test.ts`).
- **`npm install`:** `ELECTRON_SKIP_BINARY_DOWNLOAD=1 PLAYWRIGHT_SKIP_BROWSER_DOWNLOAD=1
  npm install --ignore-scripts --no-audit --no-fund`. **Headless sweep:** `npm run sim -- 181 8`.
