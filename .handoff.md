## 4. FAILED PATHS / GOTCHAS (save the next agent time)

- **`advanceMonth` = 1440 ticks, not 30.** `monthlyUpdate()` fires every 30 days; each day
  is 1440÷30 = 48 ticks (REGION_MINUTES_PER_TICK=30, MINUTES_PER_DAY=1440). A 30-tick run
  is less than 1 day — `updateSectors` never fires → sector outputs are 0.
- **`foundColony` option key is `pref`, NOT `terrain`.** The opts signature:
  `{ pref?: 'river-valley' | 'coastal' | 'highlands' | 'surprise' }`.
  Seeds 3..42 don't produce coastal starts without the pref; seed 2 + `pref: 'coastal'` does.
- **`_tileYieldCache` is transient.** Do NOT add it to serialize/deserialize. Cache is rebuilt
  on first `tileYieldFor` call — determinism is preserved because terrain is static (seed-deterministic).
- **Phase C is an intentional re-baseline.** Headless markers WILL differ from the slice-3 base
  (the prior "byte-identical" baseline). Don't "fix" the deviation — it's the terrain shaping
  economies. The headless suite still passes (stable + finite + bounded), just at shifted values.
- **Phase C activates slice 3's dormant coupling.** When towns specialise (agri town produces no
  industry goods and vice-versa), `localGoodsScarcity` > 0, and cost-push + output drag fire.
  After Phase C lands, re-run the economy-balance tests with a playtest (depression ~1929, war).
  The gate-driven (not stock-magnitude) design stays correct — don't switch it.
- **Two price notions, deliberately distinct.** ARBITRAGE price (`localGoodPrice`) = stock/demand
